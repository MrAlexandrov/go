# Family Tree Relationship Calculator (Go Implementation)

Программа для вычисления родственных связей на основе семейного дерева.

## Описание

Это Go-реализация программы для вычисления родственных связей, портированная с C++. Программа использует горутины (goroutines) для параллельной обработки данных, что делает её более эффективной при работе с большими семейными деревьями.

## Особенности реализации

### Использование горутин (Goroutines)

Программа использует трёхэтапную архитектуру с максимальной параллелизацией:

1. **Этап 1: Парсинг людей** (`parser.ParsePeopleFile`):
   - Каждый человек создаётся в отдельной горутине
   - Все люди обрабатываются полностью параллельно
   - Нет зависимостей между людьми на этом этапе

2. **Этап 2: Парсинг связей** (`parser.ParseConnectionsFile`):
   - Сначала все браки обрабатываются параллельно (каждый в своей горутине)
   - Затем все родительские связи обрабатываются параллельно
   - Браки должны быть созданы до детей, но внутри каждой группы полный параллелизм

3. **Этап 3: Парсинг определений отношений** (`parser.ParseRelationsFile`):
   - Каждая строка определения отношения парсится в отдельной горутине
   - Полностью параллельная обработка

4. **Поиск родственников** (`main.ProcessRelationsConcurrently`):
   - Каждое отношение (отец, мать, дедушка и т.д.) проверяется в отдельной горутине
   - Все результаты собираются параллельно

### Потокобезопасность

- `FamilyTree` использует `sync.RWMutex` для безопасного доступа из нескольких горутин
- Чтение данных защищено `RLock()`, запись - `Lock()`

## Формат входных данных

### people.txt

Содержит только список людей с их полом:
```
# Люди (Имя (Пол))
Иван (М)
Мария (Ж)
Алексей (М)
```

### connections.txt

Содержит браки и родительские связи:
```
# Браки (супруг <-> супруг)
Иван <-> Мария

# Родительские связи (родитель -> ребенок)
Иван -> Алексей
Мария -> Алексей
```

### relations.txt

Формат: `ПУТЬ (термин_мужской|термин_женский)`

Символы пути:
- `P` - родитель (Parent)
- `C` - ребёнок (Child)
- `S` - супруг (Spouse)
- `H` - муж (Husband)
- `W` - жена (Wife)

Примеры:
```
P (отец|мать)
C (сын|дочь)
PP (дедушка|бабушка)
PC (дядя|тётя)
```

## Сборка и запуск

### Использование Makefile (рекомендуется)

```bash
# Сборка проекта (бинарник будет в директории build/)
make build

# Сборка и запуск
make run

# Очистка артефактов сборки
make clean

# Установка зависимостей
make deps

# Запуск тестов
make test
```

### Прямая сборка через Go

```bash
# Сборка в директорию build/
go build -o build/family-tree .

# Запуск собранного бинарника
./build/family-tree

# Или напрямую без сборки
go run main.go
```

**Примечание**: Все артефакты сборки помещаются в директорию `build/`, которая игнорируется Git. Это позволяет не засорять репозиторий бинарными файлами.

Программа запросит имя человека и выведет всех его родственников.

## Преимущества Go-реализации над C++

1. **Встроенная поддержка конкурентности**: Горутины легче в использовании, чем потоки C++
2. **Максимальная параллелизация**: Разделение на три файла позволяет обрабатывать данные полностью параллельно на каждом этапе
3. **Автоматическое управление памятью**: Сборщик мусора избавляет от ручного управления памятью
4. **Простота синхронизации**: `sync.WaitGroup`, `sync.Mutex` проще в использовании
5. **Меньше кода**: Go более лаконичен при сохранении читаемости
6. **Встроенное тестирование**: Стандартный пакет `testing` для unit-тестов

## Производительность

Благодаря трёхэтапной архитектуре и горутинам:
- **Этап 1**: Все люди создаются одновременно (O(1) вместо O(n))
- **Этап 2**: Браки и дети обрабатываются параллельно внутри групп
- **Этап 3**: Все определения отношений парсятся одновременно
- **Поиск**: Все типы родственников проверяются параллельно

Это даёт значительное ускорение на больших семейных деревьях.
